```text
以下に、出願に関する資料（特許請求の範囲、明細書、必要に応じて補正書や図面説明）を添付します。
あなたは、添付資料の内容のみを根拠として、この出願の技術内容を正確に解説してください。

【最重要ルール】
1. 一般論や外部知識による補完は避け、必ず添付資料に記載された内容のみに基づいて説明すること。
2. 添付資料に記載がない事項は推測せず、「資料上明示されていない」と明記すること。
3. 解説にあたっては、請求項番号、段落番号、図番号など、根拠箇所をできるだけ明示すること。
4. 特に「課題」と「解決手段」を重視し、それぞれを明確に区別して説明すること。
5. 単なる要約ではなく、技術的な意味、請求項との対応関係、発明のポイントがわかるように整理して説明すること。
6. 複数の資料（明細書、請求項、補正書など）がある場合は、最新の請求項や補正内容を優先しつつ、明細書の記載との対応関係を示すこと。

【やってほしいこと】
以下の構成で出力してください。

# 1. 出願の概要
- この出願が何に関する発明か
- どのような技術分野の発明か
- 全体として何を実現しようとしているのか

# 2. 背景と従来技術
- 添付資料に記載された背景技術を整理すること
- 従来技術や前提技術がどのようなものであるかを、資料の記載に沿って説明すること
- そのうえで、本件発明がどのような場面を想定しているかを示すこと

# 3. 課題
- この発明が解決しようとしている課題を明確に抽出すること
- 課題が複数ある場合は分けて整理すること
- 必ず根拠となる段落番号等を示すこと

# 4. 解決手段
- 上記課題に対して、発明がどのような構成・制御・処理によって解決しようとしているのかを説明すること
- 課題と解決手段の対応関係が分かるように整理すること
- 必ず根拠となる請求項番号、段落番号、図番号等を示すこと

# 5. 請求項ごとの解説
- 各請求項について、請求項ごとに「何を特定しているか」を説明すること
- 独立請求項と従属請求項の関係が分かるように説明すること
- 従属請求項については、どの限定が追加されているかを明確にすること
- 請求項の文言をそのまま繰り返すだけでなく、技術的意味を噛み砕いて説明すること

# 6. 請求項と明細書の対応関係
- 各請求項の主要な構成要件について、明細書中の対応箇所を示すこと
- 可能なら「請求項の構成要件」「対応する明細書の記載」「補足説明」の3列相当の整理で示すこと
- 対応箇所は段落番号、図番号、実施形態などを用いて具体的に示すこと

# 7. 技術的ポイント
- この発明の技術的な肝がどこにあるかを説明すること
- 特に、従来技術との差分として重要そうな点を、添付資料に基づいて整理すること
- 発明者が重要視していると読める点があれば、それも示すこと

# 8. 図面・実施形態の役割
- 図面や実施形態がある場合、それぞれが何を説明しているかを整理すること
- どの図がどの請求項やどの技術事項の理解に役立つかを示すこと
- 図に基づく説明を行う場合は、必ず図番号を明示すること

# 9. まとめ
- この出願の発明の本質を簡潔にまとめること
- 特に「どの課題に対し」「どのような手段で」「何が特徴なのか」を簡潔に総括すること

【出力上の注意】
- 必ず添付資料ベースで記述すること
- 一般的な特許実務論、一般的な通信理論、一般的な技術常識の長い説明は不要
- 記載の薄い箇所は、無理に膨らませず「資料上の記載は限定的」と明示すること
- 文章は、技術者・知財担当者が読んで理解しやすいよう、正確かつ整理された日本語で記載すること
- 重要箇所では、請求項番号、段落番号、図番号を積極的に引用すること
```
